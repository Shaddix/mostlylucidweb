mostlylucid.Markdig.FetchExtension v1.0.0
=========================================

Initial release of the Markdig Fetch Extension - a comprehensive solution for
fetching remote Markdown content and generating table of contents.

KEY FEATURES
------------

FETCH EXTENSION:
* Simple inline syntax: <fetch markdownurl="URL" pollfrequency="24h"/>
* Multiple storage backends: In-Memory, File-based, and custom implementations
* Smart caching with configurable poll frequency (hours)
* Stale-while-revalidate pattern: Returns cached content if fetch fails
* Link transformation: Automatically rewrites relative links to absolute URLs
* Fetch metadata summaries: Display when content was last fetched
* Flexible template system for metadata display
* Full dependency injection support
* Thread-safe operations across all storage providers
* Preprocessing architecture ensures fetched content flows through your entire
  Markdig pipeline with all custom extensions
* Documentation mode: Add disable="true" to prevent tag processing, perfect for
  writing documentation about the extension itself

TABLE OF CONTENTS (TOC) EXTENSION:
* Simple inline syntax: [TOC], [TOC:2-4], [TOC:2-4:my-class]
* Automatically generates hierarchical table of contents from headers
* Configurable heading level range (e.g., H2-H4)
* Custom CSS class support for styling
* Semantic HTML5 nav element output
* Automatic anchor link generation for all headings
* Integrates seamlessly with the fetch extension

FLEXIBLE TIME UNITS:
* Poll frequency now supports multiple time units: seconds (s), minutes (m), hours (h), days (d)
* Examples: "30s", "5m", "12h", "7d"
* Backwards compatible - plain numbers still default to hours
* Prevents large numbers like "1000000000s" by using appropriate units

STORAGE OPTIONS
---------------

* In-Memory: Perfect for demos, testing, simple apps (no persistence)
* File-Based: Single-server production with disk persistence
* Database Plugins: SQLite, PostgreSQL, SQL Server (separate packages)
* Custom: Implement IMarkdownFetchService for specialized needs

USAGE EXAMPLES
--------------

Fetch Extension:
  services.AddInMemoryMarkdownFetch();
  FetchMarkdownExtension.ConfigureServiceProvider(serviceProvider);
  var pipeline = new MarkdownPipelineBuilder()
      .UseAdvancedExtensions()
      .Use<FetchMarkdownExtension>()
      .Build();

  In markdown: <fetch markdownurl="https://example.com/doc.md" pollfrequency="24h"/>

TOC Extension:
  var pipeline = new MarkdownPipelineBuilder()
      .UseAdvancedExtensions()
      .UseToc()
      .Build();

  In markdown: [TOC] or [TOC:2-4] or [TOC:2-4:my-toc-class]

Combined Usage:
  var pipeline = new MarkdownPipelineBuilder()
      .UseAdvancedExtensions()
      .Use<FetchMarkdownExtension>()
      .UseToc()
      .Build();

* .NET 9.0+
* Markdig 0.42.0+
* Microsoft.Extensions.DependencyInjection 9.0+
