Just noticed this on [Stefano Demiliani's blog](http://dotnetjunkies.com/WebLog/demiliani/posts/6132.aspx), lots and lots of bugs, some of them pretty interesting! I don't know just how 'verified' lots of them actually are:

http://www.photo.net/photo/pcd1660/bear-fight-slide-2-71.tclAn interesting idea of [Dejan Jelovic](mailto:dejan@jelovic.com). This page represents a registry of known .NET bugs. You can use it to see if the strange thing you are seeing is a known bug, or to report a new bug that you just found. 


## Known .NET Bugs 

| <!----> | <!----> | <!----> | <!----> | <!----> |
| --- | --- | --- | --- | --- |
| ID | Description | Probability\* | Severity\* | Fixed |
| 1 | **Form refuses to close** <br>If a form A contains control B that contains control C that contains control D, and D is removed while it has the focus, the user won't be able to close the form. [Details](http://www.jelovic.com/weblog/e41.htm). | High | High | 1.1 |
| 2 | **SoapFormatter and BinaryFormatter don't serialize some object graphs properly** <br>If three or more classes have the same name but are in different namespaces, are part of the same hierarchy, and contain variables that have the same name, they will not be serialized properly. [Details](http://www.jelovic.com/weblog/e42.htm). | Low | High | 1.1 |
| 3 | **Text in the Windows Forms combo box cannot be selected using the mouse** <br>Mouse selection simply does not work in a Windows Forms combo box. [Details](http://www.jelovic.com/weblog/e43.htm). | High | Low | 1.1 |
| 4 | **Arrays of structures that implement ISerializable do not deserialize correctly** <br>[Details](http://www.jelovic.com/weblog/e44.htm). | High | High | 1.1 |
| 5 | **Windows.Identity doesn't work under Windows 98** <br>On Windows 98, You cannot use the Windows.Identity to retrieve the user login name, it returns a system level token and a null user name is returned when you access the .Name Property. [Details](http://www.jelovic.com/weblog/e48.htm). | Low | Medium | No |
| 6 | **C# optimizer produces bad code when a try-finally block is inside if-else** <br>If a *try-finally* block is the last thing inside the *if* block of an *if-else* statement, the code inside the *else* block is always executed when compiling in Release mode. [Details](http://www.jelovic.com/weblog/e49.htm). | High | High | 1.1 |
| 7 | **Structs within structs cannot be deserialized** <br>If you have a struct A inside struct B, and If a struct A is inside struct B which is inside object C, deserializing is going to throw a SerializationException.  [Details](http://www.jelovic.com/weblog/e50.htm). | Medium | Low | 1.1 |
| 8 | **Problems removing an ActiveX control from its parent** <br>If an ActiveX control is removed from its parent, any action that makes the parent invisible, such as closing the form on which it is on, throws an exception.  [Details](http://www.jelovic.com/weblog/e51.htm). | Medium | High | 1.1 |
| 9 | **Font.GdiCharSet hardcoded to 1** <br>The property GdiCharSet of the font class is hard-coded to 1.  [Details](http://www.jelovic.com/weblog/e54.htm). | Low | Low | No |
| 10 | **.NET Runtime leaks large chunks of memory** <br>If you allocate a block of memory larger than some treshold, the .NET garbage collector will never reclaim it. [Details](http://discuss.develop.com/archives/wa.exe?A2=ind0202A&amp;L=DOTNET&amp;D=0&amp;P=11999). | High | High | 1.1 |
| 11 | **Instances of NumericUpDown and DomainUpDown are leaked** <br>It seems that the NumericUpDown and DomainUpDown controls forget to remove a listener to a global event, thus always staying locked in memory. [Details](http://www.automatedqa.com/techpapers/net_allocation_profiler.asp). | High | High | No |
| 12 | **SoapFormatter chokes on some strings** <br>It seems SoapFormatter cannot handle strings that contain characters in the range 0x01 - 0x1f. [Details](http://www.jelovic.com/weblog/e57.htm). | High | High | No |
| 13 | **FileStream becomes unusable when writing to a removable drive** <br>When writing to a FileStream on a removable drive, and the FileStream gets out of space, it sometimes gets in a state where no other method can be called on the stream, not even Dispose. [Details](http://www.jelovic.com/weblog/e65.htm). | Medium | Medium | No |
| 14 | **Mixed-mode assemblies can deadlock** <br>C++ .NET apps can deadlock on startup. [Details](http://msdn.microsoft.com/visualc/techinfo/articles/loaderlock/default.asp). | Low | High | No |
| 15 | **ComboBox Data Binding Bug** <br>ComboBox doesn't update the data through data binding when its style is set to DropDownList. [Details](http://www.jelovic.com/weblog/e107.htm). | Medium | Medium | No |
| 16 | **Each Windows Form leaks two GDI handles** <br>[Details](http://www.jelovic.com/weblog/e118.htm). | High | High | No |
| 17 | **Alocation fails for certain sizes** <br>If you try to allocate an array with a size range between 0x027fefbd and 0x027fffec, the framework throws exceptions. This range corresponds to memory block of little under 40MB. [Details](http://radio.weblogs.com/0106747/2003/07/11.html#a382). | Medium | High | No |
| 18 | **No way to access Environment.HasShutDownStarted** <br>This property is not static even though it should be. | Low | Low | 1.1 |
| 19 | **Thread.Abort cannot abort suspended threads** <br>If a thread is suspended, trying to abort will not work and the CPU will be pegged to 100 percent. [Details](http://www.jelovic.com/weblog/e126.htm). | Low | High | No |
| 20 | **Control.Width and Height are silently truncated to 32,767** <br>If you attempt to set a control's Width or Height property to a number larger than 32,767 it will be truncated. [Details](http://www.jelovic.com/weblog/e127.htm). | Low | Low | No |
| 21 | **ListView.Cursor set property has no effect.** <br>[Details](http://www.jelovic.com/weblog/e129.htm). | Low | Low | No |
| 22 | **RegistryKey.SetValue does not store large integers as DWORD** <br>Call RegistryKey.SetValue with an integer over 2,147,483,647 will result in the value being stored as string, and not as a DWORD. [Details](http://weblogs.asp.net/tmeston/posts/10021.aspx). | Low | Medium | No |

\* *Probability* represents the probability that you will encounter this bug. *Severity* measures how much the bug hurts when it happens.
