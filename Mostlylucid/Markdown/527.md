# Catching all errors in an ASP.NET application

<datetime class="hidden">2004-05-12T00:00</datetime>
<!-- category -- mostlylucidcouk, Imported, ASP.NET, C#, Software Development -->

Okay, I realise that probably everyone and his / her mother knows this already, but if you want to log ALL errors in an ASP.NET application, the simplest way I've found to do it is in the Global.asax.cs (or whatever codebehind)...simply do this:

```

protected void Application_OnError(object sender, EventArgs e)
	{
		Exception ex = Server.GetLastError();
		if ( ex is HttpException )
		{
			if (((HttpException)ex).GetHttpCode() == 404)
				return;
		}     
		ExceptionManager.Publish(ex);

	}
```

In this instance, ExceptionManager is the [Microsoft Exception Manager application block](http://msdn.microsoft.com/library/default.asp?url=/library/en-us/dnbda/html/emab-rm.asp)...you can (and really should) use this in conjunction with CustomError. The reason I do it here is that I prefer to make my error page flat html...just in case something really bad has happened and an active error page would cause an error itself :-)
