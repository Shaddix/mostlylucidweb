# .NET Passport on Windows Server 2003

<datetime class="hidden">2004-02-20T00:00</datetime>
<!-- category -- mostlylucidcouk, Imported, Passport, .NET, Windows Server, Security, SDK -->

There's an odd issue when you try to use Passport on Windows Server 2003 - basically, the default 'Secure Level' of Passport is 10, unfortunately if you don't use SSL this means Passport won't work correctly for you. In .NET this manifests itself as the Passport scarab showing you as logged in but if you try Context.Identity.User.IsAuthenticated, it'll always return 'false'.
Now, I know what you're thinking, just change it to '0'...umm...nope, the default Passport Administration tool installed with Windows Server 2003 won't let you change this...and you can't (well not in any way that I've found so far) upgrade to 2.5 (appears to...but the admin utility doesn't change...). So, you have two options (well three, but ignoring it rarely works...), either change the registry key ([described here](http://gotdotnet.com/Community/MessageBoard/Thread.aspx?id=144698)) - or connect to your server from a machine with [Passport SDK 2.5](http://msdn.microsoft.com/library/en-us/passport25/NET_Passport_VBScript_Documentation/NET_Passport_Fundamentals/Installation.asp) installed (from the admin tool) and change it that way. 
I have to say, the support for Passport is pretty rotten - as is the SDK and support on Server 2003 (which really is unacceptable IMHO).
